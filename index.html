Setup Instructions

Connecting to webODM

This viewer can display outputs from webODM processing. Here's how to use it:

1. Orthographic Maps

  • After processing drone imagery in webODM, navigate to your task
  • Copy the tile server URL from the task's orthophoto viewer
  • The URL format is: https://[webodm]/api/projects/[project_id]/tasks/[task_id]/orthophoto/tiles/{z}/{x}/{y}.png
  • Paste it into the "Tile Server URL" field in the Orthographic Maps tab
  • You may need to add your authentication token as a query parameter

2. 3D Models

  • Download the textured model or point cloud from your webODM task
  • Supported formats: OBJ (textured mesh), PLY (point cloud), LAS/LAZ (point cloud)
  • Use the file upload in the 3D Models tab to load the model
  • Or provide a direct URL to the model file

3. CORS Configuration

If loading data from a remote webODM instance, ensure CORS is properly configured:

  • Your webODM server must allow requests from this viewer's domain
  • Add appropriate Access-Control-Allow-Origin headers
  • Alternatively, host this viewer on the same domain as webODM

4. Local Setup

To run this viewer locally:

  • Clone the repository
  • Serve the files using any HTTP server (e.g., python -m http.server)
  • Open in your browser and start loading your webODM outputs
